## Mid 5 — 验证技术细节

### Q6: "Qt 4 线程模型里 HObject 跨线程怎么传？"
- **出题人验证**: Qt 跨线程 + HALCON 线程安全双重知识
- **能答**: ✅ Qt::QueuedConnection 自动深拷贝 + CLAUDE.md 第 36 行明确写 "HALCON 的 HImage 不是线程安全的" → 走信号槽不直接共享 → **掌握**

### Q7: "你的二进制帧 dataLen 是 BE，CRC 是 LE，为什么不一致？"
- **出题人验证**: 是否清楚 Modbus 协议字节序约定
- **能答**: 🟡 **要解释清楚**：CRC-16/MODBUS 的 LE 是 Modbus RTU 协议**原生约定**（首字节低位），dataLen 是自定义 16-bit BE。**理由**：兼容已有 Modbus 设备。
- **不补会被穿**: 如果说"忘了为什么"，面试官会怀疑代码是 copy paste。

### Q8: "HALCON 标定的 pixelSize 5.5e-6 硬编码，怎么应对不同型号相机？"
- **出题人验证**: 标定参数化能力
- **能答**: 🔴 **当前是 demo 级硬编码**（Review 2 已 flag）。需要立刻改成 settings.json 配置项。
- **诚实答**: "现在是 demo 级硬编码，因为做的时候只有一台测试相机。**v1.3 计划提到 settings.json 的 `calibration.camera_pixel_size_um` + UI 标定向导第一步采集**。"

### Q9: "ONNX CPU 39.9ms 是怎么测的？低端 CPU 测过吗？"
- **出题人验证**: 部署性能测试覆盖度
- **能答**: 🟡 测的是开发机（应该是 i7 类）。**没测过 i3/工业 IPC**。
- **诚实答**: "开发机是 i7-12 代，i3 / Atom 工业 IPC 没测过。**预计 4-6x 慢**，关键场景需要重做 benchmark。"

### Q10: "Metal U-Net IoU 0.413 vs YOLO mAP 0.745，是不是 U-Net 选错了？"
- **出题人验证**: 多模型对照实验设计
- **能答**: ✅ Metal Insight #4 明确写了"多类联合训练负迁移"，**U-Net 单类 vs 多类的对比 ablation 都跑过**。0.413 是多类下界，单类 inclusion 可能 0.6+。
- **进一步加分**: "U-Net 不是用来打败 YOLO，是用来给 30 张人工标注做像素级对照基准，证明 YOLO bbox 在小目标上的几何误差是 N 像素。"

## Bot 5 — 开放性 / 软技能

### Q11: "为什么选 YOLOv8 不选 YOLOv5 / YOLOv10？"
- **答**: YOLOv5 训练老熟悉，YOLOv8 Ultralytics 同一作者 + 更好的 ONNX 导出；YOLOv10 论文新但 NEU-DET 这类小数据集没明显优势。

### Q12: "为什么用 HALCON 不用 OpenCV？"
- **答**: HALCON 标定 + Shape Model 工业精度比 OpenCV 强 0.5-1 个数量级（亚像素拟合 + 多尺度匹配），是项目目标"工业级"必需。OpenCV 4.8 在 Robot 项目里只做相机采集和图像预处理。

### Q13: "PyQt6 vs C++ Qt，metal 项目要不要重写成 C++？"
- **答**: 不重写。metal 的瓶颈在 ONNX 推理（39.9ms），不是 UI；Python + PyQt6 开发效率 3-5 倍，且 ONNX 模型在 C++ 里也是同样的 onnxruntime。**重写无 ROI**。

### Q14: "21 个 bug 是怎么发现的？AI 协助多少？"
- **答**: DEVLOG 里写了：架构/设计层用 Claude（AI assistant）做 review，编码用 Claude Code 半自动，**所有 bug 都是人工迭代调试发现**——AI 不会主动报告 "你这里 logical bug"。AI 协作的最大价值是**写完后立刻 review 自己刚写的代码**，缩短反馈环。

### Q15: "下一步加什么功能？"
- **答**: 
  - Robot: ① 真机接 UR3e 做精度测试，② queryStatus 完整实现，③ 标定漂移检测；
  - Metal: ① 用 Defect-Spectrum 大数据集训练更强模型，② TensorRT GPU 部署 vs ONNX CPU 对照。
